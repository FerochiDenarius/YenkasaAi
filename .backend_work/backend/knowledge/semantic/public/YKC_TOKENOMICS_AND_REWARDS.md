# YKC Tokenomics and Reward Economy

## Role of YKC

Yenkasa Coin, or YKC, is the reward-oriented ecosystem currency used to connect user activity with platform growth.

The reward philosophy is:

- reward participation
- reward engagement
- reward watch time and interaction
- reward creator contribution
- align ecosystem growth with monetizable activity

## Legacy tokenomics specification

The older algorithm documentation defines this planned allocation model:

- total supply: 100,000,000 YKC
- user rewards pool: 70,000,000 YKC
- development team: 10,000,000 YKC
- company reserves: 15,000,000 YKC
- liquidity and exchange: 5,000,000 YKC

This should be treated as a design specification and roadmap reference unless separately confirmed as live production economics.

## Legacy activity reward schedule

The older mobile application algorithm document defines these base rewards and caps:

- likes: 1 YKC each, up to 50 per day
- comments: 2 YKC each, up to 20 per day
- follows: 3 YKC each, up to 10 per day
- posts: 5 YKC each, up to 5 per day, typically for verified users
- daily streak reward: 10 YKC, with higher streak bonuses
- ads viewed: 1 to 5 YKC each, with daily limits
- post views: 0.5 YKC each, with daily limits
- time spent: 1 YKC per 10 minutes, with daily limits
- daily user cap: 250 YKC
- monthly user cap: 7,500 YKC

## Verification multiplier model

The legacy document also defines reward multipliers tied to trust and activity progression:

- unverified: 1.0x
- details verified: 1.2x
- app verification phase 1: 1.5x
- phase 2: 1.8x
- phase 3: 2.2x
- phase 4: 2.7x
- phase 5: 3.3x
- phase 6: 4.0x

## Utility direction

Planned or documented YKC utility includes:

- premium ad posting
- featured content promotion
- business profile verification
- analytics or advanced service access
- custom community creation
- developer API access
- transfers and live gifting

## Current implementation direction

Current engineering notes show YKC working as:

- a transaction ledger plus balance system
- a reward path connected to posts, engagement, account activity, and gifting
- a wallet history and analytics mechanism
- a creator-economy primitive rather than just a cosmetic point system

## Blockchain roadmap interpretation

The older algorithm documentation describes a staged path:

- centralized in-app economy first
- hybrid bridge stage later
- eventual native blockchain direction

Treat the blockchain model as roadmap guidance unless a live deployment state is explicitly retrieved.
